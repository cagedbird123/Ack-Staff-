Cloud sync — use from multiple devices

Optional. Off by default — without this set up, everything stays local to this device exactly as before. Turning it on links this device to a free Google Firebase database so the same data stays in sync across every computer that connects with the same details below. Still works offline: changes save locally first and sync automatically once there's a connection.

One-time setup (about 5 minutes, free):
1. Go to console.firebase.google.com and create a project.
2. In that project, open Build → Firestore Database and click "Create database" (test mode is fine to start).
3. Click the gear icon → Project settings → scroll to "Your apps" → add a Web app.
4. Copy the apiKey, authDomain, projectId, and appId values shown there into the fields below.
5. Invent a School Code — any unguessable word or phrase, e.g. "greenview-4821". Enter the exact same code on every device that should share this data.

This device is connected to Cloud Sync.

Setup Code to share — give staff this one code (or the link below) to join from another device, instead of the five fields above. Hidden by default — click Show before reading it aloud or retyping it; Copy works either way.

Security note: the School Code works like a shared password — anyone with it and these Firebase details can read and write this data. Don't post it publicly. This is convenient sync for internal school use, not encryption or per-user login; ask if you'd like stronger access control layered on later.
